## ‚ú® Why OKLCH Format?

### 1. **Uniform Luminance**
Two colors with the same `L` value appear equally bright to the human eye‚Äîcritical for accessibility.

```
oklch(52% 0.12 260)  // Same brightness as...
oklch(52% 0.14 85)   // ...different hue, same perceived lightness
```

### 2. **Harmonious Color Relationships**
By maintaining consistent `Hue (H)` and adjusting only `Lightness (L)`, you create perfect harmony:

```
Primary:    oklch(52% 0.12 260)   // Full saturation
Light:      oklch(68% 0.12 260)   // Same hue & saturation, just lighter
Muted:      oklch(52% 0.04 260)   // Same hue & lightness, less saturated
```

### 3. **Accessibility Built-In**
WCAG contrast ratios are predictable:

| L Difference | Contrast Ratio | WCAG Level |
|--------------|----------------|-----------|
| 20% | ~4.5:1 | AA ‚úÖ |
| 30% | ~7:1 | AAA ‚úÖ |
| 50% | ~15:1 | AAA+ ‚úÖ |

### 4. **Browser Support**
```css
/* All modern browsers support oklch() natively */
color: oklch(52% 0.12 260);  /* ‚úÖ Chrome 111+, Firefox 113+, Safari 17.2+ */
```

## üìä OKLCH Format Syntax

```
oklch(L C H)

L = Lightness (0% = black, 100% = white)
C = Chroma (saturation, 0 = gray, 0.4 = highly saturated)
H = Hue (0-360¬∞, using standard color wheel)
```

**Examples:**
- `oklch(50% 0.12 260)` = 50% bright, moderate saturation, 260¬∞ hue (blue)
- `oklch(80% 0.15 85)` = 80% bright, highly saturated, 85¬∞ hue (gold)
- `oklch(20% 0.01 265)` = 20% bright, very muted, 265¬∞ hue (dark blue-gray)
